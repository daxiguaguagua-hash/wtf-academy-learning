# 本目录协作规则

## 交流方式

- 和用户说话尽量简洁明了。
- 汇报优先使用表格、流程图和高密度结论。
- 避免重复已经确认过的上下文。

## 学习陪跑规则

- 当前 Solidity 学习主线工具为 `Foundry`。
- 如果后续开始学习 `Hardhat`，必须与 `Foundry` 分开记录，避免混淆。
- 每一章默认按以下顺序推进：
  1. 讲最小知识点
  2. 让用户亲手改代码
  3. `forge build`
  4. 部署到本地链
  5. 用 `cast call` / `cast send` 实操验证
  6. 出检查题

## 代码注释规则

- 当用户在总结阶段 `@` 某个代码文件时，必须先读取该文件再总结。
- 在不破坏用户原有注释的前提下，根据当前章节知识点，为主要变量和函数补充简洁注释。
- 新增注释应以“帮助当前学习理解”为目标，避免无意义的逐行注释。
- 如果用户没有 `@` 文件，不默认批量改代码注释，除非用户明确要求。

## 进度记录规则

- 每完成一个关键学习节点，必须同时更新：
  - `学习笔记.md`
  - `交接记录.md`
- `学习笔记.md` 记录：
  - 知识点
  - 命令
  - 易错点
  - 当前章节结论
- `交接记录.md` 记录：
  - 当前进度
  - 已完成章节
  - 当前工具路线
  - 下一步任务
- 不要等用户提醒才记录，默认主动记录。

## Git 结构规则

- 当前仓库采用“外层单仓库统一管理”模式。
- `hello-web3` 作为普通目录管理，不再保留独立 `.git`。
- 以后如果在本目录下新建新的 `Foundry` 学习项目，若该项目属于当前学习总仓库，默认去掉项目自带 `.git`，避免嵌套仓库。
